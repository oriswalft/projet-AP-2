Analyse des besoins

Afin de satisfaire les besoins du projet, nous avons dû concevoir une page de connexion, qui permet donc aux utilisateurs, que ce soit des visiteurs ou des comptables, de se connecter à la base de données, et d’accéder à leurs données. Les visiteurs peuvent donc enregistrer leurs fiches de frais, donc enregistrer le nombre de nuitées, la distance (en Km), le prix du Km en fonction du type de véhicule et le repas du midi. Ils peuvent également ajouter des frais hors forfait. Les comptables eux peuvent voir les fiches de frais, et les valider.

Comment lancer l'application ?

Pour lancer l'application, il est nécessaire d'importer une copie de la base de donnée fournie sur le github puis de l'importer sur votre serveur.
Dans le fichier projet_ap_javafx > src > main > java > com > example > Partie SQL > CoBDD.java,
Modifier l'adresse IP qu'elle corresponde à celle de votre serveur, ou machine. Un utilisateur "gsb" avec des droits en SELECT, UPDATE, DELETE et INSERT dans votre SGBD est nécessaire au fonctionnement de l'application. Le code pour créer cet utilisateur est joint ci dessous : 

  
CREATE USER 'gsb'@'localhost' IDENTIFIED BY '';
GRANT SELECT, DELETE, INSERT, UPDATE ON  projet_ap2* TO 'gsb'@'localhost';
